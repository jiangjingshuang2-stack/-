# 数据集与实验设计说明

本文档用于帮助组员快速理解项目使用了哪些数据、为什么设计这些实验，以及每项实验应该观察什么。最终结果统一位于：

```text
results/final_rerun_20260606_172023/
```

## 一、研究问题

项目求解的 LASSO 问题为：

```text
min_x  0.5 * ||Ax - b||_2^2 + lambda * ||x||_1
```

- 第一项衡量模型拟合误差。
- `L1` 正则项鼓励系数变为零，从而获得稀疏模型。
- `lambda` 控制拟合误差与稀疏度之间的权衡。
- `rho` 是 ADMM 的算法参数，主要影响收敛速度。

比较的四种算法为 Subgradient、ISTA、FISTA 和 ADMM。sklearn Lasso 仅作为正确性参考，不替代自实现算法。

## 二、数据集介绍

### 1. 合成稀疏回归数据

最终主实验参数：

```text
m=80, n=200, sparsity=10, sigma=0.05, seed=42
```

- `m=80`：观测样本数。
- `n=200`：特征维度。
- `sparsity=10`：真实系数中仅有 10 个非零元素。
- `sigma=0.05`：加入观测的高斯噪声标准差。

生成流程：

1. 生成 `80 × 200` 的高斯随机设计矩阵。
2. 对每一列归一化，减少特征尺度差异。
3. 随机选择 10 个位置生成非零真实系数。
4. 按 `b = Ax* + noise` 生成观测。

合成数据的优势是已知真实系数，可以直接计算 recovery error 和 support recovery。它主要用于算法机制与超参数分析。

### 2. Diabetes

- 来源：`sklearn.datasets.load_diabetes`
- 任务：根据 10 个数值特征预测疾病进展指标。
- 总样本数：442
- 训练集：353 个样本，10 个特征
- 测试集：89 个样本，10 个特征

预处理：

- 按 `80%/20%` 划分训练集和测试集。
- 仅使用训练集统计量标准化特征和目标变量，避免测试集信息泄漏。

作用：作为稳定、规模较小的真实回归基准，验证各算法的预测误差和收敛效率。

### 3. Wine Quality Red

- 来源：项目内的 UCI Wine Quality 红酒数据文件。
- 任务：根据理化指标预测红酒质量评分 `quality`。
- 总样本数：1599
- 训练集：1279 个样本，11 个特征
- 测试集：320 个样本，11 个特征

预处理：

- 删除目标列 `quality`，其余列作为特征。
- 按 `80%/20%` 划分数据。
- 使用训练集统计量标准化特征和目标变量。

作用：验证结论能否从 sklearn 内置数据扩展到另一个本地真实回归数据集。

### 4. Communities and Crime

- 来源：项目内的 UCI Communities and Crime 数据文件。
- 任务：预测社区暴力犯罪率 `ViolentCrimesPerPop`。
- 原始样本数：1994
- 原始属性数：128
- 清洗后特征数：100
- 训练集：1595 个样本，100 个特征
- 测试集：399 个样本，100 个特征

清洗流程：

1. 使用 `communities.names` 恢复列名。
2. 将 `?` 识别为缺失值。
3. 删除 `state`、`county`、`community`、`communityname` 和 `fold` 等非预测字段。
4. 删除缺失比例超过 30% 的特征。
5. 使用训练集中的中位数填补剩余缺失值。
6. 使用训练集统计量完成标准化。

作用：验证算法在含缺失值、需要清洗且维度较高的复杂真实回归任务上的表现。

## 三、评价指标

### 合成数据指标

- `final_objective`：最终 LASSO 目标函数值。
- `recovery_error`：估计系数与真实系数之间的欧氏距离。
- `nnz`：绝对值大于 `1e-6` 的非零系数数量。
- `support_ok`：估计支撑集是否与真实支撑集完全一致。
- `iterations`：算法完成的迭代次数。
- `runtime`：运行时间。
- `primal_res`、`dual_res`：ADMM 的原始和对偶残差。

### 真实数据指标

- `train_mse`：标准化训练集上的均方误差。
- `test_mse`：标准化测试集上的均方误差。
- `nnz`：模型保留的非零特征数。
- `iterations`、`runtime`：优化效率指标。

真实数据没有已知的真实系数，因此不计算 recovery error 或 support recovery。

## 四、最终实验介绍

### 1. 合成数据基础实验

目录：`synthetic_baseline/`

目的：在同一数据和参数下比较四种算法的解质量、稀疏性与收敛速度。

主要问题：

- 四种算法是否收敛到相近目标值？
- 哪种算法迭代次数最少？
- 哪种算法能产生有效稀疏解？

### 2. Lambda 敏感性实验

目录：`lambda_sensitivity/`

测试值：

```text
0.01, 0.02, 0.05, 0.1, 0.2, 0.5, 1.0
```

目的：观察正则化强度如何影响恢复误差、目标函数和 nnz。

主要问题：lambda 增大后解是否更稀疏，以及何时出现过度收缩。

### 3. ADMM Rho 敏感性实验

目录：`rho_sensitivity/`

测试值：

```text
0.1, 0.5, 1.0, 2.0, 5.0, 10.0
```

目的：观察 rho 对 ADMM 最终解和收敛速度的影响。

主要问题：不同 rho 是否得到相同解，以及哪一范围收敛最快。

### 4. 多随机种子重复实验

目录：`repeat_experiment/`

随机种子：

```text
0, 1, 2, 3, 4
```

目的：验证主实验结论是否依赖某一次随机生成的数据。

主要问题：不同算法的恢复误差、nnz 和迭代次数是否稳定。

### 5. sklearn Lasso 对照实验

目录：`sklearn_compare/`

目的：使用成熟库验证自实现算法结果是否正确。

参数换算：

```text
alpha = lambda / n_samples
```

原因是 sklearn 的目标函数包含样本数归一化，而本项目的目标函数不包含。

### 6. 三个真实数据实验

目录：`real_diabetes/`、`real_wine_red/` 和 `real_communities_crime/`

目的：比较四种算法在实际回归任务中的预测性能、模型稀疏性与收敛效率。

主要问题：

- 不同优化算法的 test MSE 是否接近？
- ADMM 是否仍具有迭代次数优势？
- 当前 lambda 下真实数据模型是否产生明显稀疏性？

## 五、报告与 PPT 使用建议

- 用合成基础实验介绍四种算法的主要差异。
- 用 lambda 图说明误差与稀疏性的权衡。
- 用 rho 残差图说明 ADMM 参数主要影响收敛过程。
- 用重复实验说明结论稳定，不是单次随机结果。
- 用 sklearn 对照验证自实现结果可信。
- 用三个真实数据集说明算法在实际回归任务中仍然有效。

各实验的具体数值结论和图片说明，请阅读最终结果对应子目录中的 `README.md`。
