## 第1页

### 页标题
研究问题与课题背景

### 页面目标
让老师和组员在第一页就明白：我们研究的是哪类优化问题、为什么选择 Lasso、这个课题和课程内容有什么关系。

### 页面必须包含的文字内容

- `研究问题`
  我们研究高维稀疏回归问题，在观测数少于特征维度时，希望从有限观测中恢复出稀疏系数向量。

- `为什么选 Lasso`
  Lasso 在拟合误差之外引入 `L1` 正则项，可以在欠定问题中恢复稀疏解，是课程中非光滑优化的典型模型。

- `与课程的关系`
  这个题目同时涉及：
  - 非光滑优化
  - 近端梯度法
  - 加速方法
  - ADMM

- `本作业的核心目标`
  比较 `Subgradient / ISTA / FISTA / ADMM` 四种算法在稀疏恢复任务中的表现，并分析参数、噪声、规模变化和真实数据效果。

### 页面必须包含的表格内容

本页不建议放大表格。  
如果想加，可以在右下角放一个极小表格，标题为：`课题关键词`

表格列：
- 关键词
- 含义

表格内容：

| 关键词 | 含义 |
|---|---|
| 稀疏恢复 | 只恢复少量非零系数 |
| 非光滑优化 | L1 项在 0 点不可导 |
| 近端方法 | 用软阈值处理 L1 |
| ADMM | 变量分裂求解 Lasso |

### 页面必须插入的图片/图表

- 放一个“问题示意图”
- 位置：右侧中上
- 图内容建议：
  - 矩阵 `A (m×n)`
  - 稀疏向量 `x*`
  - 观测 `b = Ax* + noise`
  - 箭头指向 `Lasso` 模型
- 图的作用：
  让大家快速理解这是一个“高维稀疏恢复”问题，而不是普通回归。

### 页面排版建议

- 左侧 55% 放 4 条主 bullet
- 右侧 45% 放问题示意图
- 如果放小表格，放在右下角
- 整体风格要干净，第一页不要放太多数字

### 演讲备注（这页应该怎么讲）

建议讲 40 到 50 秒。

讲法：
1. 我们做的是高维稀疏回归问题。
2. 在 `m<n` 时，普通最小二乘不适合，所以引入 Lasso。
3. Lasso 的意义在于既拟合数据，又通过 `L1` 正则诱导稀疏解。
4. 这个题目和课程中的近端方法、ADMM 都直接对应。

## 第2页

### 页标题
Lasso 模型与四种算法

### 页面目标
这一页要把模型公式和四种算法讲清楚，让听众理解后面实验比较的对象到底是什么。

### 页面必须包含的文字内容

- `Lasso 模型`
  在页面中央明确写公式：
  `min_x 1/2 ||Ax-b||_2^2 + λ||x||_1`

- `各项含义`
  - 第一项：拟合误差
  - 第二项：L1 正则，鼓励稀疏
  - `λ`：控制拟合与稀疏之间的平衡

- `四种算法的角色`
  - `Subgradient`：baseline，直接处理非光滑项
  - `ISTA`：近端梯度法
  - `FISTA`：ISTA 的加速版本
  - `ADMM`：变量分裂法，是本作业的核心方法

### 页面必须包含的表格内容

表格标题：`四种算法在本作业中的定位`

| 算法 | 核心思想 | 优点 | 预期不足 |
|---|---|---|---|
| Subgradient | 次梯度下降 | 实现简单 | 收敛慢、步长敏感 |
| ISTA | 梯度下降 + 软阈值 | 适合 L1 问题 | 速度一般 |
| FISTA | 加速近端梯度 | 比 ISTA 更快 | 仍需较多迭代 |
| ADMM | 变量分裂 + 交替更新 | 适合光滑+非光滑结构 | 需要调 `rho` |

### 页面必须插入的图片/图表

本页不建议放实验结果图。  
建议放一个“算法关系示意框图”：
- 左边：`Subgradient`
- 中间：`ISTA -> FISTA`
- 右边：`ADMM`
- 在图旁标注：
  - “baseline”
  - “近端方法”
  - “分裂方法”

### 页面排版建议

- 上方居中放公式
- 左下放文字 bullet
- 右下放算法表格
- 算法关系示意图可以穿插在表格上方或左下角

### 演讲备注（这页应该怎么讲）

建议讲 50 到 60 秒。

讲法：
1. 先解释公式。
2. 再说明四种算法不是随便选的，而是代表三类思路。
3. 强调 `Subgradient` 是 baseline，`ADMM` 是重点，`ISTA/FISTA` 是 Lasso 中最典型的近端方法。

## 第3页

### 页标题
实验设计与数据集设置

### 页面目标
说明实验并不是随便跑的，而是按“合成数据主实验 + 参数分析 + 真实数据验证”的逻辑组织起来的。

### 页面必须包含的文字内容

- `实验分两类数据`
  - 合成数据：主实验
  - 真实数据：补充验证

- `为什么用合成数据`
  因为合成数据知道真实稀疏解 `x*`，可以直接计算恢复误差和支撑集恢复情况。

- `为什么还要用真实数据`
  因为真实数据可以验证方法在实际回归任务中的预测性能和收敛速度，避免结论只停留在模拟数据上。

- `整体实验结构`
  - 小规模合成数据基线实验
  - 中等规模正式主实验
  - `lambda` 敏感性分析
  - `rho` 敏感性分析
  - 噪声实验
  - 多随机种子重复实验
  - 三档规模对比实验
  - 正则化路径
  - 相变分析
  - 真实数据实验

- `为什么真实数据实验少于合成数据实验`
  这不是实验做少了，而是因为两类数据承担的任务不同：
  - 合成数据知道真实稀疏解，所以可以做恢复误差、支撑集恢复、相变分析、噪声实验和规模扩展实验
  - 真实数据不知道真实系数，所以主要用于验证预测误差、稀疏性和收敛效率

| diabetes | 真实数据 | 小规模真实回归验证 |
| wine_red | 真实数据 | 第二个真实回归验证 |
| Communities and Crime | 真实数据 | 高维、带缺失值、需清洗的复杂真实回归验证 |

告诉老师：我们不仅得到了主结果，还系统研究了 `lambda`、`rho` 和噪声的影响。

### 页面必须包含的文字内容

#### 第一部分：lambda 敏感性

- `lambda = 0.01, 0.05, 0.1, 0.2, 0.5`
- 小 `lambda`：误差较低但解偏稠密
- 大 `lambda`：更稀疏但容易过度收缩
- 正式主实验取 `lambda = 0.2`

#### 第二部分：rho 敏感性

- 仅对 ADMM 做分析
- `rho` 对最终解影响小，但对收敛速度影响大
- `rho = 1` 迭代次数最少

#### 第三部分：噪声实验

- 固定主参数，只改变 `sigma`
- 噪声越大，恢复误差越大
- 噪声越大，恢复出的解越不稀疏

- ADMM 收敛速度更有优势

2. 再用相变分析说明噪声与恢复成功率之间的关系
3. 特别强调：相变分析经过了“初版不理想 -> 修正设置 -> 得到可解释结果”的过程
4. 最后讲真实数据，说明方法不是只在模拟数据上成立
5. Communities and Crime 要强调“复杂”和“需要清洗”，不要只强调它的维度

## 给组员的最终提醒

这套 PPT 不是“把所有实验结果都堆上去”，而是围绕 4 个核心逻辑展开：

1. 我们研究的是一个典型的高维稀疏优化问题
2. ADMM、ISTA、FISTA 都比 Subgradient 更适合做 Lasso
3. 参数、噪声、规模都会影响恢复效果和收敛速度
4. 这些结论不仅在合成数据上成立，也在真实数据上得到了补充验证

补充提醒：
- 合成数据实验更多，是因为它承担“机制分析”的任务
- 真实数据实验更少，是因为它承担“应用验证”的任务
- 这不是设计缺陷，而是实验分工本来就不同
